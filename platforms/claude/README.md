# Claude Projects 셋업 가이드

Claude Pro / Team 사용자는 Projects 기능을 통해 5분 안에 시작할 수 있다.
무료 사용자는 일반 대화에서 시스템 프롬프트를 첫 메시지로 붙여 쓰는 방식으로 가능 (아래 "무료 사용자" 섹션).

## Claude가 다른 점

Claude는 **자연어 트리거**에 강하고, **Artifact**로 결과물을 분리해 출력할 수 있다.
이 시스템 프롬프트는 그 강점에 맞춰 재단되어 있다.

- "오토라이팅 시작" 외에 "시작하자", "오늘 머릿속 꺼내고 싶어" 같은 자연스러운 표현도 인식한다.
- 3F 정리 결과나 디코딩 결과를 **Artifact로 분리**해 출력 → 사용자가 그대로 복사해서 노션·메모·옵시디언으로 옮기기 좋다.

## 1. 새 프로젝트 만들기

Claude 좌측 메뉴에서 **"Projects" → "New project"** 클릭.

- **Name**: `오토라이팅 & 3F/5F 회고`
- **Description**: `생각과 감정을 검열 없이 꺼내고, 3F/5F로 디코딩해서 다음 행동으로 연결하는 셀프코칭 공간.`

## 2. 시스템 프롬프트 붙여넣기

`system-prompt.md`의 코드 블록 내용을 복사해서 프로젝트 **"Custom instructions"** 영역에 그대로 붙여넣는다.

## 3. 지식 파일 업로드

`knowledge-files/` 안의 3개 파일을 프로젝트의 **"Project knowledge"** 에 업로드:

- `01-methodology.md`
- `02-emotion-words.md`
- `03-templates.md`

## 4. 첫 대화

프로젝트 안에서 새 대화를 열고 다음을 입력한다.

```text
오토라이팅 시작.

오늘은 이 프로젝트의 첫 기록이야.
먼저 10분 동안 검열 없이 쓸 수 있게 시작 문장 3개를 제안해줘.
```

### 일반적인 평일 흐름

1. **"오토라이팅 시작"** → 시작 문장 받기 → 10–30분 자유 기록
2. **"3F 정리"** (또는 "이거 분리해줘" 같은 자연어) → 원문 붙여넣기 → Claude가 Artifact로 3F 결과 출력
3. Claude가 묻는 분기 질문: **"3F로 닫을까요, 5F까지 갈까요?"**
   - "3F로 닫기" → 핵심 문장 1개 + 관찰 포인트 1개로 마무리
   - "5F까지" → Future Action(가장 작은 행동 1~3개) + Feedback까지 Artifact로 확장

### Artifact 활용 팁

3F / 5F / 주말 디코딩 결과는 Artifact로 나오므로:
- **복사 버튼**으로 한 번에 복사해서 노션·옵시디언·메모 앱에 옮길 수 있다
- 매 기록마다 **날짜와 주제**가 헤더에 들어가므로 나중에 찾기 쉽다
- 같은 대화 안에서 Artifact를 여러 번 만들면 이력이 쌓인다

## 5. 캘린더 등록

`../../calendar/blurbs.md`의 평일/주말 문구를 캘린더에 등록.

## 무료 사용자: 일반 대화로 쓰기

Claude 무료 플랜이면 Projects를 못 쓰지만, 다음 방법으로 시작할 수 있다.

1. 새 대화를 열고 첫 메시지로 `system-prompt.md`의 코드 블록 내용을 그대로 붙여넣고, 이어서 "오토라이팅 시작"이라고 보낸다.
2. 감정 단어나 템플릿이 필요하면 그때그때 `docs/` 안의 해당 파일을 첨부한다.
3. 대화 하나를 "메인 기록 대화"로 정해 그 안에서 모든 데일리 기록을 이어가면 컨텍스트가 유지된다.

## 운영 리듬 (요약)

- **평일**: 3F 기본, 그날그날 5F까지 확장할지 선택
- **주말**: 한 주 기록을 모아 "주말 디코딩"
- **월간**: 주간 디코딩을 모아 "월간 리뷰"

자세한 방법론은 [docs/methodology.md](../../docs/methodology.md), 분기 가이드는 [docs/faq.md](../../docs/faq.md).
